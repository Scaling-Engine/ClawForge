<task type="auto">
  <name>Task 2: Enhance health check with HTTP verification and Slack notification</name>
  <files>.github/workflows/rebuild-event-handler.yml</files>
  <action>
Replace the existing "Health check" step (lines 93-107) in `rebuild-event-handler.yml` with an enhanced version that:

1. Waits 15 seconds for containers to stabilize (keep existing behavior).
2. For each container (clawforge-noah, clawforge-ses), does TWO checks:
   a. Docker health status via `docker inspect` (existing behavior)
   b. HTTP health check: curl `http://localhost:80/api/ping` INSIDE the container using `docker exec $CONTAINER curl -sf http://localhost:80/api/ping` (containers expose port 80 internally, Traefik routes externally — so we exec into the container to curl localhost).
3. If ANY container fails either check, set `FAILED=1`.
4. After the loop, if `FAILED=1`:
   a. Source the `.env` file from `${{ steps.setup.outputs.dir }}/.env` to get `SLACK_BOT_TOKEN` and `SLACK_OPERATOR_CHANNEL` (same vars used by the app's alerts.js).
   b. If both vars are set, send a Slack message via curl to `https://slack.com/api/chat.postMessage` with the auth header `Authorization: Bearer $SLACK_BOT_TOKEN`, posting to `$SLACK_OPERATOR_CHANNEL`. Message text: "Deploy alert: Not all ClawForge containers are healthy after rebuild. Check VPS."
   c. If vars are not set, just print a warning (non-fatal, same pattern as alerts.js).
   d. Exit with code 1 to mark the workflow step as failed.

Important: Use `set +e` before the container loop so individual curl failures don't abort the script. Re-enable strict mode after.
  </action>
  <verify>

  </verify>
  <done>Health check step curls the app endpoint inside each container, sends Slack notification on failure using SLACK_BOT_TOKEN/SLACK_OPERATOR_CHANNEL from .env, and exits non-zero on unhealthy state.</done>
