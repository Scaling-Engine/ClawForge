## Integration Report

### Complete Flows (3/4)

1. **Slack/Telegram -> Docker Job -> Notification**: User message -> agent creates job -> Docker dispatch -> container runs -> waitAndNotify -> Slack/Telegram notification. End-to-end wired.

2. **Context Hydration in Job Containers**: dispatchDockerJob -> entrypoint reads GSD_HINT -> selects agent file -> reads STATE.md/ROADMAP.md/git history -> gates on complexity -> assembles prompt. End-to-end wired.

3. **Warm/Cold Start with Volume Cache**: dispatchDockerJob creates volume -> mounts /repo-cache -> entrypoint detects .git -> fetch (warm) or clone (cold) -> flock mutex -> cp to /job. End-to-end wired.

### Partial Flow (1/4)

4. **Docker Job -> LangGraph Memory**: Notification path works, but `addToThread()` is not called in the Docker `waitAndNotify` path. The agent won't remember Docker job outcomes in subsequent conversation turns. The Actions path in `api/index.js:303` does call `addToThread`.
