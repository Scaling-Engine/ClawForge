  <action>
Add two new exported functions and modify dispatchDockerJob:

1. **volumeNameFor(instanceName, repoUrl)** -- Derives deterministic volume name.
   - Extract slug from repoUrl: strip trailing `.git`, take last path segment
   - Return `clawforge-${instanceName}-${slug}`
   - Export this function (used by entrypoint integration tests later)

2. **ensureVolume(name)** -- Creates volume if it doesn't exist.
   - Call `docker.getVolume(name).inspect()` -- if it resolves, volume exists
   - On error (404), call `docker.createVolume({ Name: name })`
   - Log creation: `console.log('Created named volume: ${name}')`
   - Do NOT export this (internal helper only)

3. **Modify dispatchDockerJob** -- Add volume mount before createContainer:
   - After computing `instanceName` (line ~56), compute volume name: `const volName = volumeNameFor(instanceName, opts.repoUrl)`
   - Call `await ensureVolume(volName)` before createContainer
   - Add `Mounts` array to HostConfig (alongside existing NetworkMode and AutoRemove):
     ```
     Mounts: [
       {
         Type: 'volume',
         Source: volName,
         Target: '/repo-cache',
         ReadOnly: false,
       },
     ],
     ```
   - Add `clawforge.volume` label to container Labels with value `volName` (for debugging/inspection)

Export volumeNameFor (useful for tests and future cleanup tooling). Keep ensureVolume unexported (internal implementation detail).
