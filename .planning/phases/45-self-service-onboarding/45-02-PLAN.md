   a. `verifyGithubPat()` — async. Fetches `https://api.github.com/user` with `Authorization: Bearer ${process.env.GH_TOKEN}`. Returns `{ success: true, login: data.login }` on 200, `{ success: false, error: 'message' }` on failure. Wrap in try/catch. 5-second AbortController timeout.

   b. `verifyDockerSocket()` — async. Import Docker from 'dockerode'. Create `new Docker({ socketPath: '/var/run/docker.sock' })`. Call `docker.ping()` wrapped in `Promise.race()` with a 5-second timeout (the research flagged this as a critical pitfall — Docker hangs if daemon is down). Return `{ success: true }` or `{ success: false, error: 'Docker daemon not reachable (timeout)' }`.

   c. `verifySlackWebhook(webhookUrl)` — async. POST to the given webhook URL with body `{ text: 'ClawForge onboarding test - connection verified' }`. Return `{ success: true }` on 200, `{ success: false, error: 'message' }` on failure. 5-second AbortController timeout.

3. Add Server Actions to `lib/chat/actions.js` at the END of the file (after the last existing export). Add these 5 new exported async functions:

   a. `verifyOnboardingGithub()` — calls `requireAuth()`, then calls `verifyGithubPat()` from `../onboarding/verify.js`. On success, calls `upsertOnboardingStep('github_connect', 'complete')` from `../onboarding/state.js`. Returns the verify result.

   b. `verifyOnboardingDocker()` — calls `requireAuth()`, then calls `verifyDockerSocket()` from `../onboarding/verify.js`. On success, calls `upsertOnboardingStep('docker_verify', 'complete')`. Returns the verify result.

   c. `verifyOnboardingSlack(webhookUrl)` — calls `requireAuth()`, then calls `verifySlackWebhook(webhookUrl)` from `../onboarding/verify.js`. On success, calls `upsertOnboardingStep('channel_connect', 'complete')`. Returns the verify result. The webhookUrl comes from the form — it's the Slack incoming webhook URL the operator configured.

   d. `dispatchOnboardingFirstJob()` — calls `requireAuth()`. Imports `createJob` from `../tools/create-job.js`. Calls `createJob()` with a minimal test job: `{ repo: process.env.GH_REPO, jobDescription: 'Onboarding verification: create a test file at onboarding-test.md confirming the agent pipeline works.' }`. On success (prUrl returned), calls `upsertOnboardingStep('first_job', 'complete')` then `markOnboardingComplete()` from `../onboarding/state.js`. Returns `{ success: true, prUrl, jobId }`. On failure, returns `{ success: false, error: err.message }`.

   e. `getOnboardingStatus()` — calls `requireAuth()`, calls `getOnboardingState()` from `../onboarding/state.js`, returns the state object. This lets the client poll for updated state.

   Add the necessary imports at the top of the file (grouped with existing imports):
   ```javascript
   import { verifyGithubPat, verifyDockerSocket, verifySlackWebhook } from '../onboarding/verify.js';
   import { getOnboardingState, upsertOnboardingStep, markOnboardingComplete } from '../onboarding/state.js';
   ```
