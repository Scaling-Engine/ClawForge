Add a `handleSend` modification: after sending in terminal mode, capture the session ID from the response header. Since `useChat` does not expose response headers directly, use `onResponse` callback in the `useChat` options:

const { messages, status, stop, error, sendMessage, regenerate, setMessages } = useChat({

Claude Code tools have names like: `Read`, `Write`, `Edit`, `MultiEdit`, `Bash`, `Glob`, `Grep`, `WebFetch`, `TodoWrite`, `TodoRead`, `_thinking`.

Add a detection set for Claude Code tool names:
```javascript
const TERMINAL_TOOL_NAMES = new Set([
  'Read', 'Write', 'Edit', 'MultiEdit', 'Bash', 'Glob', 'Grep',
  'WebFetch', 'TodoWrite', 'TodoRead', '_thinking',
]);
```

3. **Route to correct component** — In the message part rendering section (where `part.type === 'tool-invocation'` is handled), check if the tool name is a terminal tool:

```javascript
// Inside the part rendering loop/switch:
if (part.type === 'tool-invocation') {
  const isTerminalTool = TERMINAL_TOOL_NAMES.has(part.toolName);
  if (isTerminalTool) {
    return <TerminalToolCall key={part.toolCallId} part={part} />;
  }
  // Existing ToolCall rendering for LangGraph tools
  return <ToolCall key={part.toolCallId} part={part} />;
}
```

This approach is safe because Claude Code tool names (PascalCase: Read, Write, Edit) and LangGraph tool names (snake_case: create_job, get_job_status) are disjoint — there is zero collision risk between the two sets.
