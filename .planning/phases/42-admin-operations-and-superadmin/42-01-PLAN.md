<task type="auto">
  <name>Task 1: Repo DB layer + Server Actions + loadAllowedRepos migration</name>
  <files>lib/db/repos.js, lib/tools/repos.js, lib/chat/actions.js</files>
  <action>
1. Create `lib/db/repos.js` — DB-backed repo CRUD using the existing `settings` table with `type: 'repos'` and `key: 'all'`:

   - `getRepos()` — reads from settings table, returns parsed array. Falls back to empty array.
   - `saveRepos(repos)` — validates and upserts the full repos array as JSON in settings (type='repos', key='all').
   - `addRepo(repo)` — validates required fields (owner, slug, name), checks slug uniqueness, appends to array, calls saveRepos.
   - `updateRepo(slug, updates)` — finds by slug, merges updates, calls saveRepos. Returns updated repo or null.
   - `deleteRepo(slug)` — removes by slug, calls saveRepos. Returns boolean.
   - `migrateReposFromFile()` — reads `config/REPOS.json` if it exists AND no repos in DB yet, imports them to DB. Called lazily on first `getRepos()`.

   Validation rules for addRepo/updateRepo:
   - `owner` required, non-empty string
   - `slug` required, lowercase alphanumeric + hyphens, 2-50 chars
   - `name` required, 1-100 chars
   - `aliases` optional array of strings
   - `dispatch` must be 'docker' or 'actions', defaults to 'docker'
   - `qualityGates` optional array of strings (shell commands)
   - `mergePolicy` must be 'auto', 'gate-required', or 'manual', defaults to 'auto'

2. Modify `lib/tools/repos.js:loadAllowedRepos()` — import `getRepos` from `lib/db/repos.js`, call it first. If it returns a non-empty array, use that. Otherwise fall back to reading the file (existing behavior). This makes the migration transparent — first call auto-migrates from file if DB is empty.

3. Add Server Actions to `lib/chat/actions.js`:
   - `getRepoList()` — `requireAdmin()`, calls `getRepos()`, returns array
   - `addRepoAction(repoData)` — `requireAdmin()`, calls `addRepo()`, returns `{success, repo}` or `{error}`
   - `updateRepoAction(slug, updates)` — `requireAdmin()`, calls `updateRepo()`, returns `{success, repo}` or `{error}`
   - `deleteRepoAction(slug)` — `requireAdmin()`, calls `deleteRepo()`, returns `{success}` or `{error}`
   - `getConfigValues()` — `requireAdmin()`, returns object of known config keys with their values (reads LLM_PROVIDER, LLM_MODEL from env as defaults, overrides from DB)
   - `updateConfigAction(key, value, isSecret)` — `requireAdmin()`, validates key against allowlist, calls setConfigValue or setConfigSecret, returns `{success}` or `{error}`
   - `getInstancesOverview()` — `requireAuth()`, returns array of instance info by reading `instances/*/config/REPOS.json` from filesystem + checking Docker containers via dockerode for active job count

   Config key allowlist for `updateConfigAction`:
   ```
   LLM_PROVIDER, LLM_MODEL, JOB_TIMEOUT_MS, ASSEMBLYAI_API_KEY, BRAVE_API_KEY,
   SLACK_REQUIRE_MENTION, AUTO_MERGE_ENABLED
   ```
