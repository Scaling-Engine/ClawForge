  <action>
1. Create `lib/superadmin/config.js`:
   - `isSuperadminHub()` — returns `process.env.SUPERADMIN_HUB === 'true'`
   - `getInstanceRegistry()` — parses `process.env.SUPERADMIN_INSTANCES` (JSON string: `[{"name":"noah","url":"https://noah.clawforge.dev","token":"xxx"}, ...]`). Returns empty array if not set or parse fails. Always includes the local instance (from `INSTANCE_NAME` env var) with `url: null` to indicate "local".
   - `getSuperadminRole(userRole)` — returns true if `userRole === 'superadmin'`
   - `getLocalInstanceName()` — returns `process.env.INSTANCE_NAME || 'default'`

2. Create `lib/superadmin/client.js`:
   - `queryInstance(instance, endpoint, params)` — if `instance.url` is null (local), imports and calls the local handler directly. Otherwise makes `fetch(instance.url + '/api/superadmin/' + endpoint, { headers: { 'Authorization': 'Bearer ' + instance.token }, ...params })`. Returns parsed JSON. Has 5-second timeout.
   - `queryAllInstances(endpoint, params)` — calls `queryInstance` for each instance in registry in parallel via `Promise.allSettled`. Returns array of `{ instance: name, data, error }` objects. Never throws — always returns partial results.

3. Create `api/superadmin.js` — API route handler registered in `api/index.js` under `GET /api/superadmin/:endpoint`:
   - Auth: validate `Authorization: Bearer <token>` header against `process.env.AGENT_SUPERADMIN_TOKEN`. Return 401 if invalid. This is machine-to-machine auth, not user auth.
   - Endpoints:
     - `GET /api/superadmin/health` — returns `{ instance: INSTANCE_NAME, status: 'online', uptime: process.uptime(), version: pkg.version }`
     - `GET /api/superadmin/stats` — returns `{ instance: INSTANCE_NAME, activeJobs: <count from getPendingDockerJobs>, lastJobAt: <most recent job timestamp>, repoCount: <from getRepos>, userCount: <count> }`
     - `GET /api/superadmin/jobs?repo=&status=&q=&limit=50` — queries jobOutcomes + jobOrigins with optional filters, returns `{ instance: INSTANCE_NAME, jobs: [...] }`. Filter by: targetRepo (contains), status (exact), q (logSummary contains). Limit 50 per instance.

   Register the route in `api/index.js` — add a handler for paths starting with `/api/superadmin/` that delegates to `api/superadmin.js`.

4. Extend auth for superadmin role:
   - In `lib/db/schema.js`: the `role` column already accepts any text, so `superadmin` just works as a value. No schema change needed.
   - In `lib/auth/index.js`: ensure the session callback includes `user.role` (it should already from Phase 32). Verify `role: 'superadmin'` flows through to the session. Add a `requireSuperadmin()` helper alongside `requireAdmin()` in `lib/chat/actions.js` that checks `user.role === 'superadmin'`.

5. Add Server Actions to `lib/chat/actions.js`:
   - `getSuperadminDashboard()` — `requireSuperadmin()`, imports `queryAllInstances` from superadmin/client.js, queries 'health' and 'stats' endpoints for all instances, merges results into `{ instances: [{ name, status, activeJobs, lastJobAt, repoCount }] }`
   - `searchJobsAcrossInstances(filters)` — `requireSuperadmin()`, queries 'jobs' endpoint across all instances, merges and sorts by createdAt desc, returns flat array with instance name attached to each job
   - `getInstanceRegistryAction()` — `requireSuperadmin()`, returns instance names from config (not tokens)
