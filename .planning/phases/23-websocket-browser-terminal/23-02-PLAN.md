<task type="auto">
  <name>Task 1: Docker exec utilities and API routes for shell spawn and git status</name>
  <files>lib/tools/docker.js, api/index.js</files>
  <action>
**lib/tools/docker.js** -- Add two new exported functions:

1. `spawnExtraShell(workspaceId, port)`:
   - Get workspace from DB, verify containerId exists
   - docker.getContainer(ws.containerId).exec() with Cmd: ['ttyd', '-W', '-p', String(port), '--ping-interval', '30', 'tmux', 'new', '-s', `tab-${port}`], Detach: true
   - exec.start({ Detach: true })
   - Return { port }
   - Max tabs limit: if port > 7685 (4 extra tabs), throw error "Maximum tabs reached"

2. `checkWorkspaceGitStatus(workspaceId)`:
   - Get workspace from DB, verify containerId exists
   - Run `git -C /workspace status --porcelain` via container.exec (AttachStdout: true, AttachStderr: true)
   - Run `git -C /workspace log @{u}..HEAD --oneline` via container.exec (same attach pattern)
   - Collect output using a helper that reads exec stream to string (use dockerode's demuxStream or manual stream reading -- match existing patterns in docker.js for log collection)
   - Return { hasUncommitted: boolean, uncommittedFiles: string[], hasUnpushed: boolean, unpushedCommits: string[], safe: boolean }
   - safe = !hasUncommitted && !hasUnpushed
   - If git log fails (no upstream set), treat as hasUnpushed = false (not an error)

**api/index.js** -- Add two new POST sub-routes using the existing regex pattern:

1. `/workspaces/:id/shell` -- POST, auth-gated
   - Extract workspaceId from URL
   - Read `port` from request body JSON (default 7682, must be 7682-7685)
   - Call spawnExtraShell(workspaceId, port)
   - Return 200 with { port }

2. `/workspaces/:id/git-status` -- POST, auth-gated
   - Extract workspaceId from URL
   - Call checkWorkspaceGitStatus(workspaceId)
   - Return 200 with git status result

Add imports for spawnExtraShell and checkWorkspaceGitStatus at top of api/index.js alongside existing workspace imports.

**templates/app/workspace/[id]/terminal.jsx** -- Client component ('use client'):
- Dynamic import xterm.js modules in useEffect to avoid SSR (import @xterm/xterm, @xterm/addon-fit, @xterm/addon-attach, @xterm/xterm/css/xterm.css)
- Props: { workspaceId, port, ticket, wsUrl, onDisconnect }
- On mount:
  1. Create Terminal instance with cursorBlink: true, fontSize: 14, fontFamily: 'Menlo, Monaco, "Courier New", monospace', theme: { background: '#1e1e2e', foreground: '#cdd6f4', cursor: '#f5e0dc' }
  2. Load FitAddon, open terminal in ref div, call fitAddon.fit()
  3. Create WebSocket to `${wsUrl}?ticket=${ticket}`, set binaryType = 'arraybuffer'
  4. Load AttachAddon with the WebSocket
  5. Listen for window resize -> fitAddon.fit()
  6. On WebSocket close/error, call onDisconnect callback
- Cleanup: remove resize listener, close WebSocket, dispose terminal
- Render: div ref with width/height 100%

**templates/app/workspace/[id]/page.jsx** -- Server component with client island:
- Import auth from lib/auth/config.js for session check
- Import requestTerminalTicket from lib/ws/actions.js
- Import getWorkspace from lib/db/workspaces.js
- Import Terminal from './terminal.jsx' (client component)

Structure:
- Validate workspace exists and is running (redirect to /workspace or 404 if not)
- Render a client wrapper component (WorkspaceTerminalPage) that manages:

**WorkspaceTerminalPage** (client component, can be in same file or terminal.jsx):
- State: tabs array [{ id, port, ticket }], activeTabIndex, showCloseWarning, gitStatus
- On mount: request initial ticket for port 7681, add as first tab
- "New Tab" button:
  1. Determine next port (7682, 7683, etc. based on existing tabs)
  2. POST to /api/workspaces/{id}/shell with { port } to spawn ttyd
  3. Request ticket for that port via requestTerminalTicket(workspaceId, port)
  4. Add tab to state
- Tab bar: render tab buttons, clicking switches activeTabIndex
- Close tab: remove from tabs array (do NOT kill ttyd -- it will die with the container)
- Terminal rendering: for each tab, render Terminal component (display:none for inactive tabs to preserve state)
- beforeunload handler:
  1. Fetch /api/workspaces/{id}/git-status
  2. If not safe, show confirmation dialog with list of uncommitted files and unpushed commits
  3. Use window.onbeforeunload for browser-native "are you sure" dialog
  4. Also show a custom modal with details when user clicks an explicit "Close Workspace" button
- Reconnect: if Terminal calls onDisconnect, show "Reconnect" button that requests new ticket and recreates the WebSocket
- WebSocket URL: construct from window.location (wss:// for https, ws:// for http) + /ws/terminal/{workspaceId}

Layout:
- Full-screen (100vw x 100vh), dark background matching terminal theme
- Top bar: workspace info (repo name, branch), tab buttons, "New Tab" button, "Close" button
- Terminal area: fills remaining space
- Close warning modal: overlaid dialog showing uncommitted files and unpushed commits with "Close Anyway" and "Cancel" buttons
