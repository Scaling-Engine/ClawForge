<objective>
Harden all browser-facing Server Actions to return a proper HTTP 401 signal when called without a valid NextAuth session.

Purpose: The existing pattern uses `throw new Error('Unauthorized')` which surfaces as a generic error on the client. Next.js 15 provides `unauthorized()` from `next/navigation` as the canonical way to signal 401 from Server Actions and Server Components — it integrates with the `unauthorized.js` boundary convention. This satisfies WEBUI-05. WEBUI-06 is verified by confirming no change was made to API-key-protected routes.

Output: Updated `requireAuth()` in `lib/chat/actions.js`, updated inline auth checks in `lib/ws/actions.js`, and a minimal `app/unauthorized.js` boundary page.
</objective>
