**Researched:** 2026-03-05
**Domain:** Shell script synchronization, `gh pr create` CLI usage

## Summary

Phase 16.1 closes a single integration gap: `docker/job/entrypoint.sh` (the deployed production file) still uses `--body "$PR_BODY"` with shell variable interpolation for PR creation, while `templates/docker/job/entrypoint.sh` (the template/reference) was already fixed to use `--body-file /tmp/pr-body.md`. The deployed container builds from `docker/job/`, so the fix in templates never reaches production.

The gap is confined to lines 267-276 of `docker/job/entrypoint.sh` -- the PR creation block. Additionally, the Dockerfile in `docker/job/Dockerfile` is missing the `/defaults/` bake step that `templates/docker/job/Dockerfile` has (SOUL.md/AGENT.md defaults for cross-repo jobs). This second diff is a separate concern and should NOT be addressed in this phase (it relates to EXEC-02, not DELIV-01).

**Primary recommendation:** Copy the `--body-file` PR creation pattern from `templates/docker/job/entrypoint.sh` lines 267-278 into `docker/job/entrypoint.sh`, replacing the current `--body "$PR_BODY"` pattern at lines 267-276.

## The Exact Gap

### Current state in `docker/job/entrypoint.sh` (DEPLOYED, lines 267-276):

```bash
    # Use custom PR body if agent created one, otherwise default
    PR_BODY="Automated job by ClawForge"
    if [ -f /tmp/pr-body.md ]; then
        PR_BODY=$(cat /tmp/pr-body.md)
    fi
    gh pr create \
        --title "clawforge: job ${JOB_ID}" \
        --body "$PR_BODY" \
        --base main || true
```

### Target state in `templates/docker/job/entrypoint.sh` (TEMPLATE, lines 267-278):

```bash
    if [ -f /tmp/pr-body.md ]; then
        gh pr create \
            --title "clawforge: job ${JOB_ID}" \
            --body-file /tmp/pr-body.md \
            --base main || true
    else
        gh pr create \
            --title "clawforge: job ${JOB_ID}" \
            --body "Automated job by ClawForge" \
            --base main || true
    fi
```

### Why `--body-file` is better than `--body "$VAR"`

| Issue | `--body "$PR_BODY"` | `--body-file /tmp/pr-body.md` |
|-------|---------------------|-------------------------------|
| Shell special chars (`$`, backticks, `!`) | Interpreted/expanded by bash | Read verbatim from file |
| Large PR bodies (>ARG_MAX) | Can exceed shell argument length limit | No limit (file I/O) |
| Multi-line markdown tables | Subject to word splitting | Preserved exactly |
| Newlines in checklists | Require careful quoting | Native preservation |

This matters specifically for DELIV-01 because instance scaffolding PR bodies contain markdown checklists with backticks, special characters, and multi-line content.

## Architecture Patterns

### File Relationship

```
docker/job/                    <-- DEPLOYED (built into container image)
  Dockerfile                   <-- COPY entrypoint.sh /entrypoint.sh
  entrypoint.sh                <-- THIS IS THE FILE TO FIX
  hooks/gsd-invocations.js

templates/docker/job/          <-- TEMPLATE (reference, scaffolding source)
  Dockerfile                   <-- Has /defaults/ bake step (EXEC-02, out of scope)
  entrypoint.sh                <-- HAS THE FIX ALREADY
  hooks/gsd-invocations.js
  defaults/SOUL.md
  defaults/AGENT.md
```

The `run-job.yml` workflow pulls a pre-built Docker image (`scalingengine/clawforge:job-latest` or custom `JOB_IMAGE_URL`). That image is built from `docker/job/Dockerfile`, which copies `docker/job/entrypoint.sh`. The template version is never used in production -- it exists for scaffolding new projects.

### Anti-Patterns to Avoid

- **Do NOT make docker/job/entrypoint.sh import or symlink to templates/** -- these are intentionally separate. `docker/` is the deployed version; `templates/` is the scaffolding source. They may diverge for valid reasons (e.g., the `/defaults/` Dockerfile difference).
- **Do NOT address the Dockerfile `/defaults/` diff** -- that is EXEC-02 scope, not DELIV-01.
- **Do NOT refactor the entrypoint structure** -- this is a surgical one-block fix, not a rewrite.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| PR body with special chars | Shell variable interpolation (`--body "$VAR"`) | `gh pr create --body-file` | Avoids all shell escaping issues |

## Common Pitfalls

### Pitfall 1: Only fixing entrypoint.sh but not verifying identical behavior
**What goes wrong:** The two code paths look different but must produce identical results in both cases (file exists, file doesn't exist).
**How to avoid:** Verify that the fallback ("Automated job by ClawForge") is preserved for the no-file case.

### Pitfall 2: Introducing whitespace/encoding differences
**What goes wrong:** Copy-paste introduces trailing whitespace, different indentation, or CRLF line endings.
**How to avoid:** Use exact content from the template. The file uses 4-space indentation within the if block.
