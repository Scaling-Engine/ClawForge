Add workspace API route handlers to api/index.js. Follow existing patterns exactly.

**Add imports at top of file:**
```javascript
import { ensureWorkspaceContainer, stopWorkspace, destroyWorkspace } from '../lib/tools/docker.js';
import { listWorkspaces, getWorkspace, updateWorkspace } from '../lib/db/workspaces.js';
```

**Add handler functions before the POST/GET exports:**

1. `handleCreateWorkspace(request)`:
   - Parse body: `{ instanceName, repoUrl, repoSlug, secrets, llmSecrets, threadId, maxConcurrent }`
   - Validate required: instanceName, repoUrl, repoSlug. Return 400 if missing.
   - Call `ensureWorkspaceContainer(body)`.
   - If throws (concurrent limit, etc.): return 409 with `{ error: err.message }`.
   - Return 201 with `{ workspace, created }`.

2. `handleListWorkspaces(request)`:
   - Get instanceName from query params: `url.searchParams.get('instance')`
   - If no instance: return 400 `{ error: 'Missing instance query param' }`.
   - Call `listWorkspaces(instanceName)`.
   - Return 200 with `{ workspaces }`.

3. `handleStopWorkspace(request, workspaceId)`:
   - Call `stopWorkspace(workspaceId)`.
   - If `!result.ok`: return 404 with `{ error: result.reason }`.
   - Return 200 with result.

4. `handleStartWorkspace(request, workspaceId)`:
   - Get workspace record via `getWorkspace(workspaceId)`.
   - If not found: return 404.
   - Call `ensureWorkspaceContainer({ instanceName: ws.instanceName, repoUrl: ws.repoUrl, repoSlug: ws.repoSlug })`.
   - Return 200 with result.

5. `handleDestroyWorkspace(request, workspaceId)`:
   - Call `destroyWorkspace(workspaceId)`.
   - If `!result.ok`: return 404 with `{ error: result.reason }`.
   - Return 200 with result.

**Add routes to POST switch:**
```javascript
case '/workspaces':              return handleCreateWorkspace(request);
```

For routes with workspace ID, add before the `default:` case:
```javascript
default: {
  // Workspace sub-routes: /workspaces/:id/stop, /workspaces/:id/start
```

  <action>
Modify config/instrumentation.js to add workspace reconciliation on startup and periodic idle timeout checking.

**After the existing `await initDocker();` call, add:**

```javascript
// Reconcile workspace containers (sync DB with Docker state after restart)
const { reconcileWorkspaces, checkIdleWorkspaces } = await import('../lib/tools/docker.js');
await reconcileWorkspaces();

// Start idle workspace timeout checker (every 5 minutes)
const IDLE_CHECK_INTERVAL_MS = 5 * 60 * 1000;
setInterval(async () => {
  try {
    const stopped = await checkIdleWorkspaces();
    if (stopped > 0) {
      console.log(`[workspace-idle] Stopped ${stopped} idle workspace(s)`);
    }
  } catch (err) {
    console.warn(`[workspace-idle] Check failed: ${err.message}`);
  }
}, IDLE_CHECK_INTERVAL_MS);
```

This ensures:
1. On startup, workspace DB records are synced with actual Docker container state (DATA-02 -- survives restarts).
2. Every 5 minutes, idle workspaces past the timeout threshold (default 30 min) are stopped (CNTR-03).

The idle check uses fire-and-forget with error catch to never crash the event handler.
