Add `ensureWorkspaceContainer(opts)` to lib/tools/docker.js. This is the primary workspace lifecycle function implementing the state machine from the research.

Parameters:
```javascript
/**
 * @param {object} opts
 * @param {string} opts.instanceName - 'noah', 'strategyES'
 * @param {string} opts.repoUrl - Full git clone URL
 * @param {string} opts.repoSlug - Short repo name (e.g. 'clawforge')
 * @param {object} [opts.secrets] - Non-LLM secrets (GH_TOKEN, etc.)
 * @param {object} [opts.llmSecrets] - LLM-accessible secrets (AGENT_LLM_ prefixed)
 * @param {string} [opts.threadId] - Originating chat thread for Phase 24
 * @param {number} [opts.maxConcurrent=3] - Max running workspaces per instance
 * @returns {Promise<{workspace: object, created: boolean}>}
 */
```

Implementation (state machine):

1. **Check existing:** Call `getWorkspaceByRepo(instanceName, repoSlug)`.
2. **If found, handle based on status:**
   - `status=running` + container running -> update lastActivityAt, return existing
   - `status=running` + container exited -> restart container via `container.start()`, update DB status+lastActivityAt, return
   - `status=running` + container missing (404 on inspect) -> deleteWorkspace, fall through to create new
   - `status=stopped` -> start container, update DB to running+lastActivityAt, return
   - `status=creating` -> treat as failed creation, deleteWorkspace, fall through to create new
   - `status=error` -> deleteWorkspace, fall through to create new
   - `status=destroyed` -> deleteWorkspace, fall through to create new
3. **Concurrent limit check:** Call `countRunningWorkspaces(instanceName)`. If >= maxConcurrent, throw Error with message: `Max concurrent workspaces (${maxConcurrent}) reached for ${instanceName}`.
4. **Generate IDs:** `const workspaceId = crypto.randomUUID(); const shortId = workspaceId.slice(0, 8);`
5. **Create DB record** with status='creating', volumeName from `wsVolumeNameFor(instanceName, shortId)`.
6. **Ensure volume** exists (reuse internal `ensureVolume` function already in docker.js).
7. **Create container** with config from research:
   - name: `clawforge-ws-${instanceName}-${shortId}`
   - Image: `opts.image || process.env.WORKSPACE_IMAGE || 'scalingengine/clawforge:workspace-latest'`
   - Env: REPO_URL, BRANCH=main, FEATURE_BRANCH=`clawforge/workspace-${shortId}`, INSTANCE_NAME, GH_TOKEN from secrets, AGENT_LLM_ secrets
   - Labels: `clawforge=workspace`, `clawforge.instance`, `clawforge.repo`, `clawforge.workspace_id`, `clawforge.created_at`
   - ExposedPorts: `{ '7681/tcp': {} }`
   - HostConfig.NetworkMode: `${instanceName}-net`
   - HostConfig.RestartPolicy: `{ Name: 'unless-stopped' }` (CNTR-02 auto-recovery)
   - HostConfig.Memory: `2 * 1024 * 1024 * 1024` (2GB)
   - HostConfig.CpuPeriod: 100000, CpuQuota: 100000 (1 CPU)
   - Mounts: volume -> /workspace
   - Healthcheck: curl -sf http://localhost:7681/ (30s interval, 5s timeout, 3 retries, 10s start period -- all in nanoseconds)

8. **SECURITY CHECK:** Before container.create, validate no Mount has Source or Target of `/var/run/docker.sock`. Throw if found.

9. **Start container**, update DB: status='running', containerId, containerName, featureBranch, lastActivityAt=Date.now().

10. **Feature branch creation (DATA-03):** After container start, wait for /tmp/.workspace-ready (poll via docker exec `test -f /tmp/.workspace-ready` with max 30s timeout, 2s interval). The entrypoint handles branch creation, but verify it succeeded by exec'ing `git branch --show-current` and checking it matches the expected feature branch. If branch creation failed, log a warning but do NOT mark as error (the workspace is still usable on whatever branch).

11. Return `{ workspace: dbRecord, created: true }`.

Wrap the entire creation flow (steps 6-10) in try/catch. On failure: attempt container removal, update DB to status='error', re-throw.

Import at top of file: `import { createWorkspace, getWorkspace, getWorkspaceByRepo, updateWorkspace, deleteWorkspace, countRunningWorkspaces, wsVolumeNameFor } from '../db/workspaces.js';`

<task type="auto">
  <name>Task 2: Stop, destroy, reconcile, and idle timeout functions</name>
  <files>lib/tools/docker.js</files>
  <action>
Add four more exported functions to lib/tools/docker.js:

**1. `stopWorkspace(workspaceId)`**
- Get workspace record. If not found or status != 'running', return `{ ok: false, reason }`.
- Get Docker container by containerId. Call `container.stop()`.
- Update DB: status='stopped', updatedAt=Date.now().
- Return `{ ok: true, workspace }`.
- Wrap Docker call in try/catch. If container not found (404), update DB to 'error'.

**2. `destroyWorkspace(workspaceId)`**
- Get workspace record. If not found, return `{ ok: false, reason: 'not found' }`.
- If containerId exists: try to stop (ignore error), then remove container with `{ force: true }`.
- Remove Docker volume by name: `docker.getVolume(workspace.volumeName).remove()`. Wrap in try/catch (volume may not exist).
- Update DB: status='destroyed'. (Do NOT delete the DB record -- keep for audit trail. Phase 24 may need it.)
- Return `{ ok: true }`.

**3. `reconcileWorkspaces()`**
- List all containers with label `clawforge=workspace`. Optionally scope to current instance via `process.env.INSTANCE_NAME`.
- For each container:
  - Look up DB record by workspace_id label. If no record: create one (orphan recovery) with status matching container state.
  - If record exists: sync status. Container running -> DB should be 'running'. Container exited -> leave as-is (RestartPolicy will handle it, but if it was stopped manually, respect that).
- For each DB record with status='running' or 'creating':
  - Try to inspect container. If 404 (missing): update DB to 'error'.
- Log summary: `Workspace reconciliation: {N} containers, {M} DB records, {K} synced, {J} orphans recovered, {L} stale records`.

**4. `checkIdleWorkspaces(idleTimeoutMs)`**
- Default `idleTimeoutMs` to `parseInt(process.env.WORKSPACE_IDLE_TIMEOUT_MS) || 30 * 60 * 1000` (30 min).
- Call `getIdleWorkspaces(idleTimeoutMs)` from workspaces.js.
- For each idle workspace: call `stopWorkspace(workspace.id)`.
- Log: `Idle timeout: stopped {N} workspace(s)`.
- Return count of stopped workspaces.

Import `getIdleWorkspaces` from workspaces.js (add to existing import).
