  <name>Task 1: Coordinator dispatch loop with label routing and cycle detection</name>
  <files>lib/cluster/coordinator.js, lib/cluster/coordinator.test.js</files>
  <behavior>
    - resolveNextRole(currentRole, label, clusterRoles) returns the correct role object when label matches a transition
    - resolveNextRole returns null when transition maps to null (terminal)
    - resolveNextRole returns null when label has no matching transition
    - checkCycleLimit(cycleMap, cycleKey, AGENT_LIMIT) returns false when under limit
    - checkCycleLimit returns true when cycle count equals AGENT_LIMIT (5)
    - runClusterLoop stops when totalIterations reaches RUN_LIMIT (15)
    - runClusterLoop stops when no next role is available (terminal transition)
    - runClusterLoop stops and marks run "failed" when cycle limit exceeded
    - readLabelFromOutbox reads label.txt from a Docker volume and returns trimmed content
  </behavior>
  <action>
    Create lib/cluster/coordinator.js with:

    1. **Constants:** `AGENT_LIMIT = 5`, `RUN_LIMIT = 15`

    2. **resolveNextRole(currentRole, label, clusterRoles):**
       - Look up `currentRole.transitions[label]`. If null/undefined, return null (terminal).
       - Find the role in clusterRoles by name. Return role object or null.

    3. **checkCycleLimit(cycleMap, cycleKey, limit):**
       - Increment count for cycleKey in cycleMap. Return true if count > limit.

    4. **readLabelFromOutbox(volumeName):**
       - Create a temporary alpine container mounting the volume, run `cat /vol/outbox/label.txt`.
       - Parse stdout, trim whitespace. Return label string or "complete" on error.
       - Use AutoRemove: true.

    5. **dispatchClusterAgent({ role, runId, agentRunId, agentIndex, volumeName, instanceName, initialPrompt, repoUrl, branch }):**
       - Build env vars for cluster entrypoint: ROLE_NAME, INBOX_DIR=/workspace/inbox, OUTBOX_DIR=/workspace/outbox, CLUSTER_RUN_ID, ROLE_SYSTEM_PROMPT_B64 (base64-encode role.systemPrompt), ALLOWED_TOOLS (role.allowedTools.join(',')).
       - If role.mcpServers and role.mcpServers.length > 0, call buildMcpConfig(role.mcpServers) and add MCP_CONFIG_JSON env var.
       - If agentIndex === 0 and initialPrompt, add INITIAL_PROMPT env var.
       - Call dispatchDockerJob with the cluster agent image, volume binding to /workspace, cluster-agent entrypoint override.
       - Wait for container to finish. Return exit code.

    6. **runClusterLoop(cluster, runId, initialPrompt, options):**
       - Options: { instanceName, repoUrl, branch, slackClient, channelId, threadTs }
       - Start with cluster.roles[0] as currentRole, agentIndex=0, totalIterations=0, cycleMap=new Map().
       - Loop while currentRole and totalIterations < RUN_LIMIT:
         a. Build cycleKey: `${agentIndex}:${currentRole.name}:${lastLabel || 'initial'}`
         b. Check cycle limit. If exceeded, update run status to "failed" with failReason, break.
         c. Create agent run in DB (createAgentRun).
         d. Ensure cluster volume.
         e. Dispatch cluster agent container. Wait for exit.
         f. Read label from outbox.
         g. Update agent run in DB (status, label, exitCode).
         h. Post Slack thread reply with agent status.
         i. Determine next role via resolveNextRole.
         j. If next role, copy outbox to next agent's inbox (copyOutboxToInbox).
         k. Increment agentIndex, totalIterations, set lastLabel.
       - After loop: update cluster run with final status and totalAgentRuns.
       - Clean up volumes on success (delete all clawforge-cluster-{runId}-* volumes).

    7. **Tests in lib/cluster/coordinator.test.js:**
       - Test resolveNextRole with matching transition -> returns role
       - Test resolveNextRole with null transition -> returns null
       - Test resolveNextRole with unknown label -> returns null
       - Test checkCycleLimit under limit -> false
       - Test checkCycleLimit at limit -> true
       - Test checkCycleLimit increments correctly across calls
       - Do NOT test runClusterLoop or dispatchClusterAgent (require Docker) — those are integration-tested manually.

    Export: resolveNextRole, checkCycleLimit, readLabelFromOutbox, dispatchClusterAgent, runClusterLoop, AGENT_LIMIT, RUN_LIMIT.
  </action>

  </verify>
