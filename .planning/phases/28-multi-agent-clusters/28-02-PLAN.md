  <action>
    Create templates/docker/cluster-agent/entrypoint.sh based on the existing job entrypoint pattern but adapted for cluster agent roles. The script must:

    1. **Environment setup:**
       - Read env vars: ROLE_NAME (default "unknown"), INBOX_DIR (default "/workspace/inbox"), OUTBOX_DIR (default "/workspace/outbox"), CLUSTER_RUN_ID, ROLE_SYSTEM_PROMPT_B64, ALLOWED_TOOLS, MCP_CONFIG_JSON, REPO_URL, BRANCH, SECRETS, LLM_SECRETS.
       - Create directories: `mkdir -p "$INBOX_DIR" "$OUTBOX_DIR" "$OUTBOX_DIR/reports"`

    2. **Repository clone:**
       - Clone REPO_URL at BRANCH into /workspace/repo (or just /workspace if no separate repo needed).
       - Set git config user.name and user.email for commits.
       - cd into the repo directory.

    3. **Secrets injection:**
       - Parse SECRETS JSON, export each key=value as env vars (same pattern as job entrypoint).
       - Parse LLM_SECRETS JSON, export each key=value as env vars.

    4. **System prompt assembly:**
       - Decode ROLE_SYSTEM_PROMPT_B64: `ROLE_PROMPT=$(echo "$ROLE_SYSTEM_PROMPT_B64" | base64 -d)`
       - Build FULL_PROMPT that includes:
         - The role system prompt
         - Instruction about INBOX_DIR and OUTBOX_DIR paths
         - Instruction to write a label to `$OUTBOX_DIR/label.txt` before exiting (one of the transition labels from the role definition)
         - Any initial prompt from stdin or INITIAL_PROMPT env var (for first agent only)

    5. **Claude Code invocation:**
       - Build --allowedTools from ALLOWED_TOOLS env var (comma-separated -> space-separated for CLI)
       - If MCP_CONFIG_JSON is set and non-empty, write it to /tmp/mcp-config.json and add `--mcp-config /tmp/mcp-config.json`
       - Run: `claude -p "$FULL_PROMPT" --allowedTools $ALLOWED_TOOLS_ARGS --output-format stream-json --verbose 2>&1`
       - NEVER use --dangerously-skip-permissions (CLST-11 hard constraint)

    6. **Post-execution:**
       - If `$OUTBOX_DIR/label.txt` does not exist, write "complete" to it as default label.
       - Commit and push any changes to the branch.
       - Exit with claude's exit code.

    Make the script executable: `chmod +x`.

    Reference the existing job entrypoint at templates/docker/job/entrypoint.sh for patterns around git clone, secrets parsing, and claude invocation. DO NOT copy the entire job entrypoint — cluster agents have different context assembly (role prompt + inbox/outbox instead of job.md).
