<task type="auto">
  <name>Task 1: Docker client and container lifecycle</name>
  <files>lib/tools/docker.js, package.json</files>
  <action>
1. Install dockerode: `npm install dockerode@^4.0.9`

2. Create `lib/tools/docker.js` with these exports:
   - `initDocker()` -- Connect to Docker via `new Docker({ socketPath: '/var/run/docker.sock' })`, call `docker.ping()`, set `dockerAvailable = true`. On failure, set `dockerAvailable = false` and log warning (fall back to Actions). After successful ping, call `reconcileOrphans()`.
   - `isDockerAvailable()` -- Returns the cached `dockerAvailable` boolean.
   - `dispatchDockerJob(jobId, opts)` -- Creates and starts a container. `opts` shape: `{ repoUrl, branch, secrets, llmSecrets, image, networkMode, instanceName }`.
     - Container name: `clawforge-job-${jobId.slice(0, 8)}`
     - Image: `opts.image || process.env.JOB_IMAGE || 'scalingengine/clawforge:job-latest'`
     - Env array: `REPO_URL`, `BRANCH` (= `job/${jobId}`), `SECRETS` (JSON string), `LLM_SECRETS` (JSON string), `DISPATCH_MODE=docker`
     - Labels: `clawforge=job`, `clawforge.job_id={jobId}`, `clawforge.instance={instanceName}`, `clawforge.started_at={ISO timestamp}`
     - HostConfig: `NetworkMode: networkMode`, `AutoRemove: false`
     - Measure `dispatchMs = Date.now() - startTime` between create and start completion.
     - **DOCK-09**: After start, log: `console.log(\`Docker dispatch ${jobId.slice(0, 8)}: container started in ${dispatchMs}ms\`)` -- this is the startup time measurement.
     - After `container.start()`, call `saveDockerJob(jobId, container.id, instanceName)` to persist tracking.
     - Return `{ container, containerId: container.id, dispatchMs }`.
   - `collectLogs(container)` -- Use `container.logs({ stdout: true, stderr: true, follow: false })`. Handle both Buffer return (convert to string) and stream return (use `docker.modem.demuxStream` with PassThrough streams to separate stdout/stderr). Return `{ stdout, stderr }`.
   - `waitForContainer(container)` -- Call `container.wait()`, returns `{ StatusCode }`.
   - `removeContainer(container)` -- Call `container.remove()` with try/catch.
   - `inspectJob(jobId)` -- Look up containerId from DB via `getDockerJob(jobId)`. If found, call `docker.getContainer(containerId).inspect()`. Return `{ running, startedAt, status, exitCode }` or null if not found. (DOCK-10)
   - `reconcileOrphans()` -- Use `docker.listContainers({ all: true, filters: { label: ['clawforge=job'] } })`. Optionally filter by `INSTANCE_NAME` env var. For each orphan: if running, `container.kill()`. Attempt `collectLogs()` for debugging. Then `container.remove({ force: true })`. Log each action. (DOCK-08)

Import `saveDockerJob` and `getDockerJob` from `'../db/docker-jobs.js'` at the top of the file.

   - Add `containerId` column to `jobOrigins`: `containerId: text('container_id')` -- nullable, only set for Docker-dispatched jobs
   - Add `notified` column to `jobOrigins`: `notified: integer('notified').notNull().default(0)` -- flag for dedup (0 = not yet, 1 = notification sent)

2. Create `lib/db/docker-jobs.js`:
   - `saveDockerJob(jobId, containerId, instanceName)` -- Update the existing jobOrigins row to set containerId. Uses `db.update(jobOrigins).set({ containerId }).where(eq(jobOrigins.jobId, jobId))`.
   - `getDockerJob(jobId)` -- Select from jobOrigins where jobId matches, return row (has containerId, dispatchMethod).
   - `markDockerJobNotified(jobId)` -- Update notified=1 on jobOrigins row.
   - `isJobNotified(jobId)` -- Check if notified=1. Used by webhook handler for dedup.
   - `getPendingDockerJobs()` -- Select jobOrigins where dispatchMethod='docker' and notified=0 and createdAt > (now - 24h). Used for reconciliation.

   Import `getDb` from `'./index.js'`, `jobOrigins` from `'./schema.js'`, and `eq` from `'drizzle-orm'`.
