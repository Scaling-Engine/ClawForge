  <files>lib/tools/repos.js, lib/ai/tools.js</files>
  <action>
1. Update `lib/tools/repos.js`:
   - `loadAllowedRepos()` already returns repo objects. No code change needed -- the `dispatch` field will naturally pass through from JSON parsing.
   - `resolveTargetRepo()` already returns the full repo object. The `dispatch` field will be available on the resolved object.
   - Add a new export `getDispatchMethod(resolvedRepo)`:
     ```javascript
     export function getDispatchMethod(resolvedRepo) {
       // Explicit per-repo config takes priority
       if (resolvedRepo?.dispatch) return resolvedRepo.dispatch;
       // Default: docker if available, actions if not
       return 'docker';
     }
     ```

2. Update `lib/ai/tools.js` createJobTool handler:

   Add imports at top:
   ```javascript
   import { isDockerAvailable, dispatchDockerJob, waitForContainer, collectLogs, removeContainer } from '../tools/docker.js';
   import { markDockerJobNotified } from '../db/docker-jobs.js';
   import { getJobOrigin } from '../db/job-origins.js';
   import { getDispatchMethod } from '../tools/repos.js';
   import { summarizeJob } from '../ai/index.js';
   import { createNotification } from '../db/notifications.js';
   import { saveJobOutcome } from '../db/job-outcomes.js';
   ```

   NOTE: `getJobOrigin` is imported from `'../db/job-origins.js'` -- this is critical for the waitAndNotify function to route notifications to the originating thread.

   After resolving the target repo and before calling `createJob()`, determine dispatch method:
   ```javascript
   const dispatchMethod = (isDockerAvailable() && getDispatchMethod(resolvedTarget) === 'docker')
     ? 'docker' : 'actions';
   ```

   Modify `saveJobOrigin` call to pass `dispatchMethod`:
   ```javascript
   saveJobOrigin(result.job_id, threadId, detectPlatform(threadId), dispatchMethod);
   ```

   **Actions path (existing, unchanged):** When `dispatchMethod === 'actions'`, call `createJob(enrichedDescription, { targetRepo: resolvedTarget })` as before. Return immediately.

   **Docker path (new):** When `dispatchMethod === 'docker'`:
   a. Still call `createJob(enrichedDescription, { targetRepo: resolvedTarget })` to create the job branch (audit trail + entrypoint reads job.md from branch).
   b. Then call `dispatchDockerJob(result.job_id, { ... })` with:
      - `repoUrl`: `https://github.com/${resolvedTarget?.owner || process.env.GH_OWNER}/${resolvedTarget?.slug || process.env.GH_REPO}.git`
      - `branch`: `job/${result.job_id}`
      - `secrets`: `process.env.AGENT_SECRETS || '{}'`
      - `llmSecrets`: `process.env.AGENT_LLM_SECRETS || '{}'`
      - `image`: `process.env.JOB_IMAGE || 'scalingengine/clawforge:job-latest'`
      - `networkMode`: `process.env.DOCKER_NETWORK || 'noah-net'`
      - `instanceName`: `process.env.INSTANCE_NAME || 'noah'`
   c. Fire-and-forget: Spawn `waitAndNotify(container, result.job_id, threadId)` as a detached async (no await). This MUST NOT block the tool return.
   d. Return the same JSON shape as the Actions path: `{ success: true, job_id, branch, dispatch_method: 'docker' }`.

3. Add `waitAndNotify` function in `lib/ai/tools.js` (private, not exported):

   This function waits for container completion, builds a notification payload matching the Actions webhook shape, routes to the originating Slack/Telegram thread, and cleans up the container.

   Key implementation details:
   - Call `waitForContainer(container)` to get `{ StatusCode }`.
   - Call `collectLogs(container)` to get `{ stdout, stderr }`.
   - Build `results` object matching the shape from `handleGithubWebhook`: `{ job, pr_url, run_url, status, failure_stage, merge_result, log, changed_files, commit_message, target_repo }`.
   - Query GitHub API for PR info: `const prs = await githubApi(\`/repos/${GH_OWNER}/${GH_REPO}/pulls?head=${GH_OWNER}:job/${jobId}&state=all\`)` -- import `githubApi` from `'../tools/github.js'`.
   - Call `summarizeJob(results)` and `createNotification(message, payload)` to reuse existing notification pipeline.
   - Call `markDockerJobNotified(jobId)` for dedup against Actions webhook.
   - Look up origin via `getJobOrigin(jobId)` (imported at top of file -- see imports above).
   - If origin found, call `saveJobOutcome(...)` and route to platform:
     - Slack: parse `origin.threadId` as `channel:threadTs`, use `WebClient.chat.postMessage`.
     - Telegram: use dynamic import of `'../tools/telegram.js'` for `sendMessage`.
   - Call `removeContainer(container)` for cleanup.
   - Wrap entire body in try/catch, with `removeContainer` in both success and error paths.

   IMPORTANT: The Telegram import from `lib/ai/tools.js` is `'../tools/telegram.js'` (not `'../../lib/tools/telegram.js'`).
