```javascript
// Inject chat context for conversational bridge (INTG-02)
if (opts.chatContext) {
  // JSON-encode to handle newlines/special chars safely; cap at 20KB
  const encoded = JSON.stringify(opts.chatContext).slice(0, 20000);
  env.push(`CHAT_CONTEXT=${encoded}`);
}
```

Use `JSON.stringify` to encode the chat context as a single-line string -- this avoids Docker env var issues with newlines and special characters (see Research pitfall 1). The workspace entrypoint can `JSON.parse(process.env.CHAT_CONTEXT)` to recover the original text.
