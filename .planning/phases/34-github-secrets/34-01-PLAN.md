    2. Create `lib/github-api.js` — GitHub Secrets + Variables CRUD wrapper (GHSEC-01):

       Import `githubApi` from `./tools/github.js`. Import `tweetnacl` and `tweetnacl-sealedbox-js`.

       Helper `base()` returns `/repos/${process.env.GH_OWNER}/${process.env.GH_REPO}`.

       `encryptForGitHub(value, publicKeyBase64)` — Sealed-box encrypt a plaintext value for GitHub's Secrets API:
       - Decode publicKeyBase64 with `Buffer.from(publicKeyBase64, 'base64')` into Uint8Array
       - Encode value with `Buffer.from(value)` into Uint8Array
       - Call `sealedbox.seal(messageBytes, publicKey)` to encrypt
       - Return `Buffer.from(encryptedBytes).toString('base64')`

       Secrets CRUD:
       - `listSecrets()` — GET `${base()}/actions/secrets`, return `data.secrets || []`
       - `getPublicKey()` — GET `${base()}/actions/secrets/public-key`, return `{ key, key_id }`
       - `upsertSecret(name, value)` — GET public key, encrypt value with `encryptForGitHub()`, PUT `${base()}/actions/secrets/${name}` with `{ encrypted_value, key_id }`. Note: PUT returns 201 (created) or 204 (updated) — both are success. The `githubApi()` wrapper calls `res.json()` which will fail on 204 (no body). Handle this by catching the error on 204 or by checking `res.status === 204`. The simplest fix: use a raw fetch for PUT that accepts 201/204 without parsing JSON, OR wrap the githubApi call in try/catch since 204 means success. Recommended: add a `{ raw: true }` option to the PUT call or handle the empty-body case. Actually, the cleanest approach: for PUT and DELETE calls that may return 204 No Content, call `githubApi()` but wrap in a try-catch that treats empty-body parse errors as success when the status was 2xx. Alternatively, make a `githubApiRaw()` variant that returns `res` without parsing. Use the simplest approach: add the fetch directly for PUT/DELETE (skip githubApi for these two methods only) using the same auth headers pattern from `githubApi`. This avoids modifying the shared helper.
       - `deleteSecret(name)` — DELETE `${base()}/actions/secrets/${name}`. Same 204 handling.

       Variables CRUD:
       - `listVariables()` — GET `${base()}/actions/variables`, return `data.variables || []`. Note: variables return `{ name, value, created_at, updated_at }` — values ARE returned (unlike secrets).
       - `createVariable(name, value)` — POST `${base()}/actions/variables` with `{ name, value }`
       - `updateVariable(name, value)` — PATCH `${base()}/actions/variables/${name}` with `{ value }`
       - `deleteVariable(name)` — DELETE `${base()}/actions/variables/${name}`

       Export all 8 functions.

    3. Add 8 server actions to `lib/chat/actions.js` (append after existing actions):

       GitHub Secrets actions:
       - `listGitHubSecrets()` — requireAuth(), dynamic import listSecrets from '../github-api.js', also fetch locally-cached masked values from config. For each secret from GitHub, check if `getConfigSecret('ghsec:${name}')` has a value — if so, extract last 4 chars for masked display. Return array of `{ name, created_at, updated_at, masked: '****XXXX' | null }`.
       - `createGitHubSecret(name, value)` — requireAuth(), validate name matches `/^[A-Z_][A-Z0-9_]*$/` AND starts with `AGENT_` (return `{ error }` if not), dynamic import upsertSecret, call it. Then store locally: `setConfigSecret('ghsec:${name}', value)` for future masked display (GHSEC-03). Return `{ success: true }`.
       - `updateGitHubSecret(name, value)` — Same as create (GitHub PUT is upsert). Update local cache too.
       - `deleteGitHubSecret(name)` — requireAuth(), dynamic import deleteSecret, call it. Also delete local cache: delete the `ghsec:${name}` config row. Return `{ success: true }`.

       GitHub Variables actions:
       - `listGitHubVariables()` — requireAuth(), dynamic import listVariables. Return the array directly (variables include values).
       - `createGitHubVariable(name, value)` — requireAuth(), validate name format, dynamic import createVariable. Return `{ success: true }`.
       - `updateGitHubVariable(name, value)` — requireAuth(), dynamic import updateVariable. Return `{ success: true }`.
       - `deleteGitHubVariable(name)` — requireAuth(), dynamic import deleteVariable. Return `{ success: true }`.

       All actions wrapped in try/catch returning `{ error: string }` on failure.

       For the delete config cache in deleteGitHubSecret, add a `deleteConfigSecret(key)` call. If `deleteConfigSecret` does not exist in config.js yet, add it — pattern: `db.delete(settings).where(and(eq(settings.type, 'config_secret'), eq(settings.key, key))).run()`.
