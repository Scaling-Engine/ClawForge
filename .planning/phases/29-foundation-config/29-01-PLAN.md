**lib/db/crypto.js** — Create the encryption module. Copy the upstream implementation verbatim from the research code examples section. Uses Node.js built-in `crypto` module (AES-256-GCM, PBKDF2 with 100K iterations). Exports `encrypt(plaintext)` and `decrypt(encryptedJson)`. Key derived from `process.env.AUTH_SECRET` directly — NEVER via `getConfig()` (circular dep). Salt: `'thepopebot-config-v1'` (keep for migration compat).

**lib/db/config.js** — Cherry-pick from upstream `https://raw.githubusercontent.com/stephengpope/thepopebot/main/lib/db/config.js`. This file provides CRUD for config values in the `settings` table using type discriminators:
- `'config'` for plain values (`getConfigValue`/`setConfigValue`)
- `'config_secret'` for encrypted values (`getConfigSecret`/`setConfigSecret`)
- `'llm_provider'` for custom provider definitions (`getCustomProvider`/`setCustomProvider`/`deleteCustomProvider`/`getCustomProviders`)

Imports:
- `{ getDb }` from `'./index.js'`
- `{ settings }` from `'./schema.js'`
- `{ encrypt, decrypt }` from `'./crypto.js'`
- `{ eq, and }` from `'drizzle-orm'`
- `{ randomUUID }` from `'crypto'`

Each function queries the `settings` table filtered by `type` and `key`. Secret values are encrypted before insert and decrypted on read. All writes set `createdAt` and `updatedAt` as `Date.now()`. Use `randomUUID()` for new row IDs.

**lib/llm-providers.js** — Cherry-pick from upstream `https://raw.githubusercontent.com/stephengpope/thepopebot/main/lib/llm-providers.js`. Exports:
- `BUILTIN_PROVIDERS` object with keys `anthropic`, `openai`, `google`. Each has `{ name, models: [{ id, name }], defaultModel }`.

**lib/config.js** — Cherry-pick from upstream `https://raw.githubusercontent.com/stephengpope/thepopebot/main/lib/config.js`, then add a `setConfig` wrapper.

Imports:
- `{ getConfigValue, getConfigSecret, setConfigValue, setConfigSecret, getCustomProvider }` from `'./db/config.js'`
- `{ BUILTIN_PROVIDERS, getDefaultModel }` from `'./llm-providers.js'`

Implementation:
1. Define `CONFIG_KEYS` set (plain config keys: LLM_PROVIDER, LLM_MODEL, SYSTEM_PROMPT, etc.)
2. Define `SECRET_KEYS` set (encrypted config keys: ANTHROPIC_API_KEY, OPENAI_API_KEY, GOOGLE_API_KEY, GH_TOKEN, etc.)
3. Define `DEFAULTS` object (LLM_PROVIDER: 'anthropic', etc.)
4. `_cache` Map with 60-second TTL
5. `getConfig(key)` — resolution chain: cache -> custom provider check -> secret DB -> config DB -> process.env -> DEFAULTS -> special LLM_MODEL handling via getDefaultModel
6. `setConfig(key, value)` — NEW wrapper not in upstream. If key is in SECRET_KEYS, call `setConfigSecret(key, value)`. Otherwise call `setConfigValue(key, value)`. Always call `invalidateConfigCache()` after write.
7. `invalidateConfigCache()` — clears the _cache Map.

IMPORTANT: Do NOT import from `lib/ai/model.js` — that would create a circular dependency risk.
