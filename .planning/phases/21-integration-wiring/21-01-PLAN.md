**Change 2 — inspectJob wiring (DOCK-10):**
1. Update the import on line 11 to add `inspectJob`: `import { isDockerAvailable, dispatchDockerJob, waitForContainer, collectLogs, removeContainer, inspectJob } from '../tools/docker.js'`.
2. In the `getJobStatusTool` handler (line 256), after `const result = await getJobStatus(job_id)` and before `return JSON.stringify(result)`, add Docker container inspection when a job_id is provided:
```javascript
// Augment with Docker container inspection if available
if (job_id) {
  try {
    const inspection = await inspectJob(job_id);
    if (inspection) {
      result.container = inspection;
    }
  } catch { /* non-fatal — container may not exist or Docker unavailable */ }
}
```
This adds a `container` key with `{ running, startedAt, status, exitCode }` to the response for Docker-dispatched jobs. For Actions-dispatched jobs, `inspectJob` returns null (no DB entry), so no extra data appears.
