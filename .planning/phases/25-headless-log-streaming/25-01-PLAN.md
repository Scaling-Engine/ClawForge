  <name>Task 1: Create stream manager singleton</name>
  <files>lib/tools/stream-manager.js</files>
  <action>
Create `lib/tools/stream-manager.js` — an in-memory singleton that tracks active Docker log streams per jobId. Store the map in `globalThis.__clawforge_streams` (same pattern as `globalThis.__clawforge_docker` in docker.js lines 18-21) to survive Next.js module reloads.

Export a single `streamManager` object with these methods:

1. `register(jobId, logCleanup)` — Adds entry: `{ logCleanup, subscribers: new Set(), startedAt: Date.now(), slackStatusTs: null, slackChannel: null }`. `logCleanup` is a function that calls `.destroy()` on the Docker log stream.

2. `subscribe(jobId, callback)` — Adds `callback` to subscribers Set. Returns an `unsubscribe` function. If jobId not found, returns no-op.

3. `emit(jobId, type, data)` — Iterates subscribers, calls each with `(type, data)`. Wrap each call in try/catch to prevent one broken subscriber from killing others.

4. `complete(jobId)` — Emits `'complete'` event with `{ elapsedMs: Date.now() - startedAt }` to all subscribers. Calls `logCleanup()`. Deletes entry from map.

5. `cancel(jobId)` — Same as `complete` but emits `'cancelled'` type instead of `'complete'`. Calls `logCleanup()`. Deletes entry.

6. `setSlackStatus(jobId, channel, statusTs)` — Stores Slack message timestamp for edit-in-place updates (STRM-06). Used by plan 03.

7. `getSlackStatus(jobId)` — Returns `{ channel, statusTs }` or null.

8. `isActive(jobId)` — Returns boolean.

Guard: If `globalThis.__clawforge_streams` already exists (hot reload), reuse it — do NOT reinitialize.
  </action>

<task type="auto">
  <name>Task 2: Create log parser with semantic event filtering and secret scrubbing</name>
  <files>lib/tools/log-parser.js</files>
  <action>
Create `lib/tools/log-parser.js` with two exported functions:

**`scrubSecrets(text)`:**
- Regex-based scrubbing of sensitive values before any SSE emission (STRM-08)
- Patterns to match and replace with `[REDACTED]`:
  - `AGENT_\w+=[^\s]+` (env var assignments like `AGENT_GH_TOKEN=abc123`)
  - `ghp_[A-Za-z0-9]{36,}` (GitHub PATs)
  - `sk-[A-Za-z0-9]{40,}` (OpenAI keys)
  - `xoxb-[A-Za-z0-9-]+` (Slack bot tokens)
  - `xoxp-[A-Za-z0-9-]+` (Slack user tokens)
  - `Bearer\s+[A-Za-z0-9._-]{20,}` (Bearer tokens)
- Return scrubbed string

**`parseLineToSemanticEvent(rawLine)`:**
- Takes a single line of container stdout (already line-split by the streaming function)
- Returns `null` for lines that should be suppressed, or a semantic event object `{ type, ...data }`
- Processing order:
  1. Run `scrubSecrets()` on rawLine first — defense in depth
  2. Try `JSON.parse(rawLine)` — handle structured JSONL
  3. For Claude Code `assistant` type: extract tool_use content blocks:
     - `Write`, `Edit`, `MultiEdit` → `{ type: 'file-change', operation: toolName.toLowerCase(), path: input.path || input.file_path }`
     - `Bash` → `{ type: 'bash-output', command: input.command?.slice(0, 120) }`
     - Text blocks > 20 chars → `{ type: 'decision', text: text.slice(0, 200) }`
  4. For Claude Code `result` type → `{ type: 'complete', subtype, result: result?.slice(0, 300) }`
  5. For GSD hooks (`event: 'gsd:wave-start'`, `gsd:task-start`, etc.) → `{ type: 'progress', label }`
  6. For `stream_event` type (partial deltas) → return `null` (suppress — these are noisy fragments)
  7. For `system` type (init) → return `null` (suppress)
  8. For unstructured text lines:
     - `[stderr]` prefix → `{ type: 'error', message: line.slice(0, 200) }`
     - Git operations (`git commit`, `git push`, `[git]`) → `{ type: 'progress', label: line.slice(0, 100) }`
     - Blank or single-char lines → `null`
     - Everything else → `null` (suppress raw noise)

All emitted event objects MUST have their string fields passed through `scrubSecrets()`.

IMPORTANT: The current entrypoint runs `claude -p` without `--output-format stream-json`, so container stdout is primarily unstructured text. Build the parser to handle BOTH formats gracefully — it should not crash on either.
