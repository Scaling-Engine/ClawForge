**CRITICAL ordering change (Research Pitfall 2):** Currently GSD_HINT is computed at Step 8c (line 132-139) AFTER system prompt assembly at Step 7 (line 85-96). The fix:

1. Move Step 8 (read job description, lines 98-102) to BEFORE Step 7. It only needs `/job/logs/${JOB_ID}/job.md` which exists after clone.

2. Move Step 8c (GSD hint derivation, lines 132-139) to right after Step 8. This gives us GSD_HINT before we need it.

3. Modify Step 7 (system prompt assembly) to select between AGENT.md and AGENT_QUICK.md based on GSD_HINT:

```bash
# Select agent instructions based on job complexity
if [ "$GSD_HINT" = "quick" ]; then
    # Try instance AGENT_QUICK.md, fall back to defaults
    if [ -f "/job/config/AGENT_QUICK.md" ]; then
        AGENT_FILE="/job/config/AGENT_QUICK.md"
    elif [ -f "/defaults/AGENT_QUICK.md" ]; then
        AGENT_FILE="/defaults/AGENT_QUICK.md"
    elif [ -f "/job/config/AGENT.md" ]; then
        AGENT_FILE="/job/config/AGENT.md"
    fi
else
    # Full agent instructions for complex jobs
    if [ -f "/job/config/AGENT.md" ]; then
        AGENT_FILE="/job/config/AGENT.md"
    fi
fi

if [ -n "$AGENT_FILE" ]; then
    SYSTEM_PROMPT="${SYSTEM_PROMPT}$(cat "$AGENT_FILE")"
fi
```

The fallback chain for quick jobs: instance AGENT_QUICK.md -> defaults AGENT_QUICK.md -> instance AGENT.md (last resort). For complex jobs: instance AGENT.md only (existing behavior).
