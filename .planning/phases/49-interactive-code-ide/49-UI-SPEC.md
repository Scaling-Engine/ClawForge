## Interaction Contract

### Tab System

- **Fixed 3 tabs:** Code (AI streaming), Shell (xterm.js terminal), Editor (file tree). Fixed order on initial render.
- **DnD reorder:** All 3 tabs are DnD-sortable via @dnd-kit. PointerSensor with `activationConstraint: { distance: 5 }` to prevent click/drag conflict.
- **Active tab tracking:** By tab ID string ('code' | 'shell' | 'editor'), not index. ID-based tracking survives reorders.
- **Default active tab:** 'shell' on initial page load (user launched an Interactive workspace — they want the terminal).
- **Tab switch:** Instant — no loading state between tabs. Content panels use CSS visibility (not unmount) to preserve xterm.js state across tab switches.

### Interactive Button (Chat Input)

- **Placement:** Inline with existing Code toggle and sub-mode dropdown in chat-input.jsx. Visible only when `codeActive === true` (Code mode must be on to launch workspace).
- **Label:** "Interactive" (text label, not icon-only — distinct from the `</>` Code toggle).
- **Loading state:** Button label changes to "Launching..." with a spinner while `launchWorkspace` Server Action is in flight. Button disabled during launch. No timeout UI — if launch fails, error toast shown.
- **On success:** Router pushes to `/code/{workspaceId}`. Chat page is left in browser history.
- **Idempotent:** If chat already has a linked running workspace (`codeWorkspaceId` FK set), button label is "Resume" and redirects directly without calling launchWorkspace again.

### Workspace Close Flow

- **Close button:** Top-right of IDE page. Labeled "Close Workspace".
- **Safe close:** If git status is clean (no uncommitted changes, no unpushed commits), close workspace immediately and redirect to `/chats`.
- **Unsafe close confirmation:** If uncommitted changes or unpushed commits exist, show an inline warning panel (not a modal dialog) within the IDE top bar. Warning lists uncommitted file count and/or unpushed commit count.
- **Warning panel CTAs:** "Close Anyway" (destructive — red) and "Cancel" (neutral). No additional confirmation required after "Close Anyway" — single confirmation only.

### Shell Tab Disconnect / Reconnect

- **Disconnected state:** Tab label shows a red dot indicator. Tab body shows a centered "Disconnected" message with a "Reconnect" button.
- **Reconnect button:** Calls `requestTerminalTicket` Server Action and re-attaches xterm.js. No page reload.

## Layout Contract

### /code/{id} Page

```
┌─────────────────────────────────────────────────────────────┐
│ TOP BAR (40px, bg #181825)                                  │
│  [repo/branch breadcrumb]   [Code][Shell][Editor] [+Tab]   │
│                             [Files] [Close Workspace]       │
├─────────────────────────────────────────────────────────────┤
│ CONTENT AREA (flex: 1, 100vh - 40px)                        │
│  ┌──────────────────────┐  ┌───────────────────────────────┐│
│  │ FILE TREE SIDEBAR    │  │ ACTIVE TAB CONTENT            ││
│  │ (240px, collapsible) │  │ (flex: 1)                     ││
│  │                      │  │                               ││
│  └──────────────────────┘  └───────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

- Page fills 100vw × 100vh. No main app sidebar or nav visible.
- File tree sidebar: 240px fixed width, collapsible via "Files" toggle button. Toggled display — not animated slide (matches existing workspace-terminal-page pattern).
- Content area: flex row. Tab content panels fill remaining width after sidebar (if visible).
- All three tab panels are rendered in the DOM simultaneously. Only the active panel is visible (`display: block` / `display: none`). This preserves xterm.js instance state across tab switches.

### Chat Input Interactive Button Placement

```
[</>] [Plan ▾] [Interactive] ··· [mic] [textarea] [send]
```

- Interactive button appears between the sub-mode dropdown and the mic button.
- Visible only when `codeActive === true` AND user is admin.
- Uses same height and padding as existing Code toggle: `px-2 py-1 text-xs font-mono rounded-lg`.

## Accessibility

- All tab buttons: `role="tab"`, `aria-selected`, `aria-controls` pointing to panel ID.
- Tab panels: `role="tabpanel"`, `aria-labelledby` pointing to tab button ID.
- Interactive button: `aria-label="Launch interactive workspace"`, `aria-busy` set to `true` during launch.
- Close Workspace button: `aria-label="Close workspace"`.
- Reconnect button: `aria-label="Reconnect terminal"`.
- File tree items: `role="treeitem"` with `aria-expanded` on directories.
- Minimum contrast: All text on IDE dark surface meets WCAG AA (4.5:1). #a6adc8 on #181825 = 5.2:1. #cdd6f4 on #1e1e2e = 10.8:1.
