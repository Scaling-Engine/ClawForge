Create `lib/chat/components/superadmin-monitoring.jsx` following the exact SuperadminDashboard pattern:

1. **File header**: `'use client';` directive. Import `useState`, `useEffect`, `useCallback` from React. Import `getMonitoringDashboard` from `'../actions.js'`.

2. **Helper functions** (copy formatRelativeTime from superadmin-dashboard.jsx or redefine):
   - `formatRelativeTime(timestamp)` -- returns "Never", "Just now", "Xm ago", "Xh ago", "Xd ago"
   - `getHealthColor(rate)` -- returns Tailwind text color class: rate === null -> "text-muted-foreground", rate >= 0.9 -> "text-emerald-500", rate >= 0.7 -> "text-yellow-500", else -> "text-destructive"

3. **StatusBadge component** (same as superadmin-dashboard.jsx): Shows online/offline with colored dot.

4. **OnboardingBadge component**:
   - If `completedAt` is truthy, show green "Complete" badge.
   - If `completedAt` is null, show yellow badge with `currentStep` value (e.g., "Step: first_job").
   - If onboarding data is null/undefined, show gray "N/A" badge.

5. **UsageBar component**:
   - Props: `used` (number), `limit` (number or null).
   - If limit is null, show "{used} jobs (no limit)" in muted text.
   - If limit is a number, show a progress bar: width = `Math.min((used / limit) * 100, 100)%`.
   - Bar color: >=90% -> bg-destructive, >=70% -> bg-yellow-500, else -> bg-emerald-500.
   - Text below bar: "{used} / {limit} jobs".

6. **MonitoringCard component** (per-instance card):
   - Props: `instance` (one item from the instances array).
   - Card layout (rounded-lg border bg-card p-4 space-y-3):
     a. **Header row**: Instance name (font-semibold) + StatusBadge.
     b. **Error row** (if instance.error): Red text with error message.
     c. **Metrics grid** (grid grid-cols-3 gap-2 text-center):
        - Error count: `instance.errorCount24h` with label "Errors (24h)"
        - Success rate: `instance.jobSuccessRate?.rate !== null ? Math.round(instance.jobSuccessRate.rate * 100) + '%' : '--'` with color from getHealthColor, label "Success Rate"
        - Jobs run: `instance.jobSuccessRate?.total ?? 0` with label "Jobs (24h)"
     d. **Usage section** (border-t pt-2): UsageBar with `used=instance.usage?.jobsDispatched ?? 0` and `limit=instance.usage?.limits?.jobsPerMonth ?? null`.
     e. **Footer row** (flex justify-between text-xs text-muted-foreground border-t pt-2):
        - OnboardingBadge with onboarding data.
        - "Last error: {formatRelativeTime(instance.lastErrorAt)}"

7. **LoadingSkeleton component**: Same pattern as superadmin-dashboard.jsx -- animated pulse cards.

8. **MonitoringDashboard (exported) component**:
   - State: `data`, `loading`, `error` (same as SuperadminDashboard).
   - `loadData` callback: call `getMonitoringDashboard()`, handle result.error and catch.
   - `useEffect`: call loadData immediately, set 30-second interval, cleanup on unmount.
   - Render:
     a. If loading -> LoadingSkeleton.
     b. If error -> red error banner.
     c. **Summary bar** (rounded-lg border bg-card p-4): "Instance Health Monitor" heading. Grid cols-3: total instances count, average success rate (computed from instances), total errors 24h (sum of errorCount24h).
     d. **Instance cards grid** (grid gap-4 sm:grid-cols-2): Map instances to MonitoringCard.
     e. Empty state if no instances: "No instances configured."
  </action>
