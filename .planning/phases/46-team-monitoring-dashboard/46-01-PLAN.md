1. **Add `getConsecutiveFailureCount(n)` to `lib/db/job-outcomes.js`** (additive -- do NOT modify existing functions):
   - Import `desc` from drizzle-orm (already imported).
   - Query the `n` most recent rows from `jobOutcomes` ordered by `createdAt DESC`.
   - Count consecutive rows where `status !== 'success'` from the start. Break at first `'success'`.
   - Return the count (number). Return 0 if no rows or first row is success.
   - Default `n = 10` to allow checking more than the threshold.

2. **Create `lib/monitoring/alerts.js`** (NEW file):
   - Import `getConfigValue`, `setConfigValue` from `../db/config.js`.
   - Import `getConsecutiveFailureCount` from `../db/job-outcomes.js`.
   - Constants: `FAILURE_THRESHOLD = 3`, `ALERT_COOLDOWN_MS = 60 * 60 * 1000` (1 hour).
   - Export `async function checkAndAlertConsecutiveFailures(instanceName)`:
     a. Call `getConsecutiveFailureCount(FAILURE_THRESHOLD)`.
     b. If count < FAILURE_THRESHOLD, return early.
     c. Read throttle: `getConfigValue('alert:consecutive_failure:' + instanceName)`. Parse as integer timestamp.
     d. If `Date.now() - lastAlertAt < ALERT_COOLDOWN_MS`, return early (throttled).
     e. Send Slack alert using the exact pattern from billing warnings: dynamic import `@slack/web-api`, check `SLACK_BOT_TOKEN` and `SLACK_OPERATOR_CHANNEL`, `postMessage` with text: `"Alert: Instance \"${instanceName}\" has ${count} consecutive job failures. Check /admin/superadmin/monitoring for details."`.
     f. On success, write throttle timestamp: `setConfigValue('alert:consecutive_failure:' + instanceName, String(Date.now()))`.
     g. Wrap the entire Slack call in try/catch. On failure, `console.warn('Failed to send consecutive failure alert:', err.message)`. Never throw -- this is non-fatal.

3. **Add `getMonitoringDashboard()` Server Action to `lib/chat/actions.js`** (additive -- add after `getSuperadminDashboard`):
   - Must be marked with `'use server'` (file already has this).
   - Call `await requireSuperadmin()`.
   - Dynamic import `queryAllInstances` from `../superadmin/client.js`.
   - Run `Promise.all([queryAllInstances('health'), queryAllInstances('usage'), queryAllInstances('onboarding')])`.
   - Merge results per instance into array:

   - Return `{ instances }`. On catch, return `{ error: 'Failed to load monitoring dashboard' }`.
