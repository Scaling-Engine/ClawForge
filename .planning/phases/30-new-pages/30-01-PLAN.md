**Server Actions (actions.js):** Add these functions at the end of the file, each starting with `await requireAuth()`:

1. `getPendingPullRequests()` — Uses dynamic import of `loadAllowedRepos` from `../tools/repos.js` and `githubApi` from `../tools/github.js`. Iterates all allowed repos via `Promise.all`, fetches `GET /repos/{owner}/{slug}/pulls?state=open&per_page=50` for each. Maps each PR to include `_repo: owner/slug`. Returns flat array. Catches errors per-repo (returns [] on failure) and catches top-level errors (returns []).

2. `getPendingPRCount()` — Calls `getPendingPullRequests()`, filters out drafts (`!pr.draft`), returns `.length`. Returns 0 on error.

3. `approvePullRequest(owner, repo, prNumber)` — Calls `githubApi(/repos/${owner}/${repo}/pulls/${prNumber}/reviews, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ event: 'APPROVE' }) })`.

4. `requestChanges(owner, repo, prNumber, body = 'Changes requested')` — Same as approve but with `event: 'REQUEST_CHANGES'` and body field.

5. `getRunners()` — Uses `githubApi` to fetch `GET /repos/${GH_OWNER}/${GH_REPO}/actions/runners` where GH_OWNER and GH_REPO come from `process.env`. Returns `data.runners || []`. Catches 403/errors gracefully, returns [].

6. `updatePassword(currentPassword, newPassword)` — Uses dynamic import of user functions from `../db/users.js`. Gets current user from `auth()` (import from `../auth/index.js`), verifies current password with `verifyPassword`, then updates. Returns `{ success: true }` or `{ error: 'message' }`. Import `auth` at top of function via dynamic import if not already imported, or use existing pattern.

Note: `'use server'` directive is already at the top of actions.js — no need to add it again.
