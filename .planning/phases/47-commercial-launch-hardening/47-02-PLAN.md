    Update docs/OPERATOR_GUIDE.md with three additions. Do NOT remove or restructure existing content — only add to it.

    **1. Add a "Troubleshooting" section** (after the Deployment section, before Current Instances Reference).

    Include the top 10 errors an operator will encounter, each with Symptom, Cause, and Fix:

    1. **Container crash-loop: "Could not find a production build"**
       - Symptom: PM2 restarts endlessly, logs show "Could not find a production build in '/app/.next'"
       - Cause: `npm run build` was not run before `docker compose up`, or .next/ is missing from the bind mount
       - Fix: Run `npm run build` on the host, then restart: `docker compose restart {service}`

    2. **Job container exits immediately (exit code 1)**
       - Symptom: Job dispatches but completes in <5 seconds with failure status
       - Cause: Missing AGENT_SECRETS or AGENT_LLM_SECRETS, or GitHub token lacks repo scope
       - Fix: Check AGENT_SECRETS JSON is valid, verify GH_TOKEN has contents:write + pull_requests:write on target repos

    3. **Slack webhook returns 401 or "invalid_auth"**
       - Symptom: Bot doesn't respond to messages, logs show signature verification failure
       - Cause: SLACK_SIGNING_SECRET doesn't match the Slack app's signing secret, or bot token expired
       - Fix: Regenerate signing secret in Slack app settings, update SLACK_SIGNING_SECRET, rebuild container

    4. **"ONBOARDING_ENABLED redirect loop"**
       - Symptom: Browser redirects infinitely between / and /onboarding
       - Cause: ONBOARDING_ENABLED=true but onboarding wizard page is not accessible (build error or missing route)
       - Fix: Check `npm run build` completed without errors. Verify templates/app/onboarding/page.js exists

    5. **Docker network not found on job dispatch**
       - Symptom: Job dispatch fails with "network {name} not found"
       - Cause: DOCKER_NETWORK value doesn't match actual Docker network name. Docker Compose prefixes with project name
       - Fix: Run `docker network ls | grep {name}` to find actual name. Update DOCKER_NETWORK env var to match

    6. **Billing limit blocks jobs unexpectedly**
       - Symptom: Job dispatch returns "Monthly job limit reached" but operator hasn't used that many jobs
       - Cause: billing_limits table has a row with a low limit, or periodMonth calculation crossed month boundary
       - Fix: Check limits via /admin/billing page. Superadmin can adjust at /admin/superadmin/billing

    7. **Sentry not capturing errors**
       - Symptom: Errors occur but nothing appears in Sentry dashboard
       - Cause: SENTRY_DSN not set, or NEXT_PUBLIC_SENTRY_DSN not set (client errors need the public DSN)
       - Fix: Set both SENTRY_DSN and NEXT_PUBLIC_SENTRY_DSN in .env, rebuild (`npm run build && docker compose restart`)

    8. **GitHub webhook not triggering notifications**
       - Symptom: Jobs complete (PR created) but no notification appears in Slack/Telegram
       - Cause: GH_WEBHOOK_SECRET mismatch, or webhook URL not pointing to {APP_URL}/api/github/webhook
       - Fix: Verify webhook in GitHub repo settings. Secret must match GH_WEBHOOK_SECRET env var exactly

    9. **"better-sqlite3 is not compatible with Edge Runtime"**
       - Symptom: Build error or runtime crash mentioning better-sqlite3 and Edge Runtime
       - Cause: A middleware or Edge function is importing a module that transitively imports better-sqlite3
       - Fix: Database operations must run in Server Components or API routes, never in middleware. Check import chains

    10. **Workspace container starts but terminal shows blank screen**
        - Symptom: Workspace opens in browser but ttyd terminal shows nothing
        - Cause: WebSocket proxy not connecting to ttyd inside the container, or container port mapping incorrect
        - Fix: Check that the workspace container is running (`docker ps`). Verify APP_URL matches the URL in the browser. Check browser console for WebSocket errors

    **2. Add v3.0 environment variables to the Environment Variables section.**

    Under a new sub-heading "### Observability & Billing Variables" (after the existing "Secret Convention" subsection), add:

    | Variable | Required | Description |
    |----------|----------|-------------|
    | `SENTRY_DSN` | No | Sentry project DSN for server-side error tracking. When unset, Sentry is disabled (zero network calls) |
    | `NEXT_PUBLIC_SENTRY_DSN` | No | Sentry DSN for client-side error capture. Must match SENTRY_DSN value |
    | `ONBOARDING_ENABLED` | No | Set to `true` to redirect new users to the onboarding wizard on first login. Remove or set to any other value to disable |
    | `SLACK_OPERATOR_CHANNEL` | No | Slack channel ID for operational alerts (billing 80% warnings, consecutive failure alerts). When unset, alerts are silently skipped |

    **3. Update the Deployment section** to remove references to `npx thepopebot init` and `npm run setup` which reference the upstream scaffolding approach that ClawForge has diverged from. Replace with ClawForge-specific instructions:

    In the "Build and Start" subsection, ensure the sequence is:
    1. Clone the repo
    2. Copy `.env.example` to `.env` and fill in values
    3. `npm install`
    4. `npm run build`
    5. `docker compose up -d`

    Do NOT modify the Docker Compose architecture diagram or the HTTPS/Let's Encrypt instructions — those are correct.
