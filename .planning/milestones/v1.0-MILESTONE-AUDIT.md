## Critical Bugs Found (E2E Testing)

### BUG-1: Split Module Context — features-context.jsx vs .js (HIGH)

**Phases affected:** 48, 49, 50
**Breaks:** Code Mode Toggle Flow, Feature Flag Gate Flow

`chat-page.jsx` imports `FeaturesProvider` from `../features-context.jsx` (source).
`chat.jsx` imports `useFeature` from `../features-context.js` (compiled artifact).
Both files exist in `lib/chat/` — Next.js treats them as separate modules, each with their own `createContext({})`. The Provider and consumer never share state.

**Result:** `useFeature('codeWorkspace')` always returns `false`. Code toggle and Interactive button are permanently hidden.

**Fix:** Delete `lib/chat/features-context.js` or change chat.jsx import to `../features-context.jsx`.

### BUG-2: Resume Button for Starting Workspaces (MEDIUM)

**Phase affected:** 49
**Breaks:** Session Continuity Flow (partial)

`getLinkedWorkspace` in chat.jsx includes `starting` status workspaces. Page.js at `/code/[id]` only accepts `running` status. Clicking "Resume" for a starting workspace causes immediate redirect back to `/chats`.

**Fix:** Narrow `getLinkedWorkspace` hydration to `running` only, or add a loading/polling state to page.js.
